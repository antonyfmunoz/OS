# Tool: Canva

## What This Tool Does

Canva is two surfaces in one product. The **GUI app** (web, desktop, mobile)
is a drag-and-drop visual editor with a template economy and an AI layer
called Magic Studio. The **Connect API** (public, GA in 2024, brand-template
ID format migrated September 2025) is a REST surface that lets external
systems generate designs from brand templates, upload assets, export to
PNG/JPG/PDF/MP4, list folders, post comments, and read brand settings.

Core capabilities:

- **Brand Template Autofill** — the headline API. Submit a JSON payload of
  `{field_name: {type, value}}` keyed against a Brand Template's dataset and
  Canva renders a personalized design. The template is designed once in the
  GUI; thousands of variants are generated by API.
- **Designs API** — create blank designs, list user designs, get metadata,
  read URLs (`urls.edit_url`, `urls.view_url`).
- **Exports API** — async export jobs (PNG, JPG, PDF Standard, PDF Print, MP4,
  GIF, PPTX). Poll until `status=success`, then download from a 24-hour
  signed URL.
- **Assets / Uploads** — upload images and videos to the user's Canva account,
  attach metadata tags, reference them inside designs.
- **Folders** — organize designs and assets, list contents.
- **Brand Templates** — list templates, fetch dataset (the schema of fillable
  fields), apply via Autofill.
- **Comments** — programmatic collaboration threads on designs.
- **Users / Me** — capability + profile lookup, capability checks.
- **Resizes, Imports** — resize a design to a new format; import external
  files into Canva designs.
- **Magic Studio (GUI only)** — Magic Design, Magic Media (Dream Lab /
  Leonardo Phoenix), Magic Write, Magic Edit, Magic Eraser, Magic Expand,
  Magic Switch (format/language conversion), Magic Morph, Background Remover,
  Brand Voice. **Not exposed via Connect API.** Anything Magic Studio is a
  GUI workflow.

## EOS Integration

Canva is the visual production layer for the brand-content half of EOS.
The work splits cleanly between human creative iteration in the GUI and
agent-driven scale via the API.

**Human (Antony) in the GUI:**
- **Personal brand hero designs** — long-form carousel templates, profile
  headers, video thumbnails. Iterate visually with Magic Studio (Magic Design
  for first drafts, Magic Edit for variant exploration, Background Remover
  for product shots).
- **Lyfe Spectrum apparel mockups** — drop a logo onto a t-shirt mockup,
  Magic Morph to test placements, export print-ready PDFs.
- **Initiate Arena funnel hero assets** — landing-page hero images, lead
  magnet covers, sales-page section graphics.
- **Brand Hub maintenance** — Brand Kit holds the canonical Lyfe Institute,
  Lyfe Spectrum, Empyrean Studio, and personal-brand color palettes, fonts,
  logos. This is the source of truth that the API reads from.

**Agents via Connect API:**
- **Social batch generator** — one Brand Template per platform (square IG,
  vertical Reel cover, LinkedIn 1200x627, X 1600x900). The
  `content_planner` skill writes daily content to Neon, the
  `canva_autofill` worker pulls the day's posts, calls `POST /v1/autofills`
  per platform with `{headline, body, cta, image_url}`, polls for the
  resulting design, calls `POST /v1/exports` for PNG, downloads, and hands
  off to the `social_publisher` skill.
- **Lyfe Spectrum product mockup variations** — one mockup template, autofill
  loop over color/size SKUs, export PNG per variant, push to Shopify or
  static product pages.
- **Initiate Arena lead-magnet personalization** — autofill `{first_name,
  niche, goal}` into a coaching workbook template, export PDF Standard,
  attach to Resend email.
- **Brand Kit drift detection** — nightly job lists Brand Templates, reads
  dataset, alerts when a template no longer references the canonical brand
  colors.

The dual-use principle: **Antony designs the hero, agents fill the
variants.** Never have an agent try to create a design from scratch — the
GUI is faster and the API has no Magic Studio anyway.

## Authentication

OAuth 2.0 Authorization Code with **PKCE (SHA-256, mandatory)**. There is
no API-key shortcut. Every integration must register at canva.dev as an
Integration, choose scopes, and walk a real user through consent.

```
Authorization endpoint: https://www.canva.com/api/oauth/authorize
Token endpoint:         https://api.canva.com/rest/v1/oauth/token
```

Flow:
1. Generate `code_verifier` (43-128 chars, base64url) and
   `code_challenge = base64url(sha256(code_verifier))`.
2. Redirect user to authorize URL with `client_id`, `redirect_uri`, `scope`,
   `code_challenge`, `code_challenge_method=S256`, `state`, `response_type=code`.
3. On callback, exchange `code` + `code_verifier` for an access token at the
   token endpoint with `grant_type=authorization_code`.
4. Use `Authorization: Bearer <access_token>`. Refresh with `grant_type=refresh_token`.

**Scopes are explicit and non-implicative.** `asset:write` does not grant
`asset:read`. Always request the matched read scope. Common scopes:

- `app:read app:write` — integration metadata
- `asset:read asset:write`
- `brandtemplate:meta:read brandtemplate:content:read`
- `design:meta:read design:content:read design:content:write`
- `folder:read folder:write`
- `comment:read comment:write`
- `profile:read`

**Brand Templates and Autofill require the calling user to be a member of
a Canva Enterprise organization.** Pro and Free accounts can list designs
and export, but cannot autofill. This is the single biggest gotcha for
solo founders — see Gotchas.

EOS storage: refresh token in Neon `tenant_secrets` table, access token in
Redis with TTL = `expires_in - 60s`, never logged.

### Connect API — autofill a brand template (curl)

```bash
# 1. Read the dataset to discover field names
curl -s https://api.canva.com/rest/v1/brand-templates/${TEMPLATE_ID}/dataset \
  -H "Authorization: Bearer ${CANVA_TOKEN}"

# 2. Submit autofill job
curl -s -X POST https://api.canva.com/rest/v1/autofills \
  -H "Authorization: Bearer ${CANVA_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "brand_template_id": "'"${TEMPLATE_ID}"'",
    "title": "IG post 2026-04-06",
    "data": {
      "headline":  {"type": "text",  "text": "Structure over discipline"},
      "subhead":   {"type": "text",  "text": "Initiate Arena"},
      "hero_image":{"type": "image", "asset_id": "'"${ASSET_ID}"'"}
    }
  }'
# → { "job": { "id": "...", "status": "in_progress" } }

# 3. Poll until success
curl -s https://api.canva.com/rest/v1/autofills/${JOB_ID} \
  -H "Authorization: Bearer ${CANVA_TOKEN}"
# → { "job": { "status": "success", "result": { "type":"create_design",
#     "design": { "id":"DAF...", "urls": { "edit_url": "...", "view_url":"..." } } } } }

# 4. Export the resulting design
curl -s -X POST https://api.canva.com/rest/v1/exports \
  -H "Authorization: Bearer ${CANVA_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{ "design_id": "'"${DESIGN_ID}"'", "format": { "type": "png" } }'

# 5. Poll export job
curl -s https://api.canva.com/rest/v1/exports/${EXPORT_JOB_ID} \
  -H "Authorization: Bearer ${CANVA_TOKEN}"
# → { "job": { "status": "success", "urls": ["https://export-download..."] } }
# Download URLs expire after 24h.
```

### Connect API — Python (httpx, no official SDK)

```python
import httpx, time, os

BASE = "https://api.canva.com/rest/v1"
H = {"Authorization": f"Bearer {os.environ['CANVA_TOKEN']}"}

def autofill(template_id: str, title: str, data: dict) -> dict:
    job = httpx.post(f"{BASE}/autofills", headers=H,
                     json={"brand_template_id": template_id,
                           "title": title, "data": data}).json()["job"]
    delay = 1.0
    while job["status"] == "in_progress":
        time.sleep(delay); delay = min(delay * 1.6, 8.0)  # exponential backoff
        job = httpx.get(f"{BASE}/autofills/{job['id']}", headers=H).json()["job"]
    if job["status"] != "success":
        raise RuntimeError(f"autofill failed: {job}")
    return job["result"]["design"]

def export_png(design_id: str) -> str:
    job = httpx.post(f"{BASE}/exports", headers=H,
                     json={"design_id": design_id,
                           "format": {"type": "png"}}).json()["job"]
    delay = 1.0
    while job["status"] == "in_progress":
        time.sleep(delay); delay = min(delay * 1.6, 8.0)
        job = httpx.get(f"{BASE}/exports/{job['id']}", headers=H).json()["job"]
    return job["urls"][0]
```

### Asset upload (multipart-less, header-encoded metadata)

```bash
curl -X POST https://api.canva.com/rest/v1/asset-uploads \
  -H "Authorization: Bearer ${CANVA_TOKEN}" \
  -H "Content-Type: application/octet-stream" \

```

## Conceptual Model

**Canva is a template economy with an AI surface bolted on top, exposed
twice: once as a GUI for humans and once as an API for systems.** Internalize
three things and the rest follows:

1. **Brand Template = schema, Design = instance.** A Brand Template is the
   only object the API can autofill. It defines named, typed slots (text,
   image, chart). A Design is the rendered output. The Connect API is
   essentially "instantiate this schema with these values, give me the
   instance, then export the instance." If you try to programmatically build
   a design from scratch, you are using the API wrong.

2. **Brand Kit is the source of truth for visual identity.** Colors, fonts,
   logos, voice, image styles all live in Brand Hub (Enterprise) or the
   Brand Kit (Pro). Both the GUI and the API draw from it. Magic Studio
   reads it for AI generations. Brand Templates inherit it. Change Brand Kit,
   templates re-render with the new values. This is why EOS keeps brand
   identity in Canva and not duplicated in code.

3. **Magic Studio is GUI-only and is the moat.** Magic Design, Magic Media
   (Dream Lab on Leonardo Phoenix), Magic Switch, Magic Morph, Magic Edit,
   Magic Write, Background Remover, Brand Voice — none have Connect API
   equivalents. The API is for *scale*; the GUI is for *creative leverage*.
   The EOS pattern is: Antony uses Magic Studio in the GUI to design one
   beautiful Brand Template, then agents call autofill to make ten thousand
   variants of it. Never expect agents to be creative in Canva — that is
   not what the API surface is for.

The async job pattern (`POST` returns `{job: {id, status: in_progress}}`,
`GET /<resource>/<job_id>` returns the final state) is universal across
autofill, export, asset upload, design import, resize. Treat it as a single
skill: submit, exponential-backoff poll, handle terminal `success`/`failed`.

## Gotchas

- **Brand Template + Autofill require Canva Enterprise membership.** Pro
  and Free accounts get a 403 on `/v1/brand-templates` and `/v1/autofills`.
  For solo founder pre-revenue this means: either upgrade to Enterprise
  (~$30/user/mo, contact sales), use a manual GUI workflow until revenue
  justifies it, or co-design templates inside an existing Enterprise tenant.
  This is the single most common Connect API blocker.
- **September 2025 Brand Template ID migration.** Old IDs still work for
  6 months from migration; after that they 404. If you stored template
  IDs before Sept 2025, refetch via `GET /v1/brand-templates` and rewrite
  your config.
- **Scopes do not imply each other.** `asset:write` without `asset:read`
  means you can upload but cannot list. `design:content:write` does not
  grant `design:content:read`. Request both. Re-consent is required when
  scopes change — old refresh tokens don't auto-upgrade.
- **Autofill field types must match exactly.** A `text` field cannot accept
  an `image` payload, and vice versa. Read the dataset first, never guess
  field names. Capitalization matters. Missing required fields return 400
  with no auto-defaulting.
- **Image autofill takes `asset_id`, not URL.** Upload to
  `/v1/asset-uploads` first, wait for that job, then pass the resulting
  `asset.id` into the autofill payload's `image` field.
- **Export jobs are async, download URLs expire in 24 hours.** Persist the
  binary into your own storage (S3, R2, Neon BYTEA) immediately. Do not
  store the Canva URL in a database expecting to use it tomorrow.
- **Rate limits are per-endpoint, per-user, per-minute.** Notable:
  `POST /v1/exports` is 20/min/user, `GET /v1/exports/{id}` is 120/min/user.
  Autofill polling at 1s with exponential backoff stays well under the cap.
  A naive `while True: get; sleep(0.1)` loop will trip 429 in seconds.
- **PDF Print vs PDF Standard.** `format.type=pdf` defaults to standard;
  for print-shop output specify `pdf_export.export_quality=pro` and CMYK
  flags. Lyfe Spectrum print orders need this — don't send Standard PDFs
  to a printer.
- **Magic Studio is not in the API.** Do not promise an agent that it can
  call Magic Design or Dream Lab. Plan workflows around: Antony designs
  hero in GUI → save as Brand Template → agent autofills.
- **OAuth callback host must match exactly.** Tailscale IPs and `localhost`
  are accepted in dev, but production redirects must be HTTPS public
  domains. EOS uses a single canonical callback at the gateway, not
  per-service callbacks.
- **No webhooks for design lifecycle events.** As of 2026-04, Connect API
  has no design.created / design.updated webhooks. If you need to know
  when Antony manually edits a design, you must poll `GET /v1/designs`
  and diff `updated_at`.
- **Comments API is one-way useful.** You can create and read comments,
  but Canva surfaces them in the GUI without push notifications back to
  your integration — they are best used as audit trail, not realtime sync.
- **Free plan cannot use the Connect API at all** for production work
  beyond the most basic profile read. Treat Pro as the practical floor
  for non-autofill use, Enterprise as the floor for autofill.
- **`design.id` format is `DAF...` and stable**, but the `urls.edit_url`
  signed URL rotates. Store the design id, regenerate edit URLs on demand.

## Conceptual Model

**Canva is a template economy with an AI surface bolted on top, exposed
twice: once as a GUI for humans and once as an API for systems.** Internalize
three things and the rest follows:

1. **Brand Template = schema, Design = instance.** A Brand Template is the
   only object the API can autofill. It defines named, typed slots (text,
   image, chart). A Design is the rendered output. The Connect API is
   essentially "instantiate this schema with these values, give me the
   instance, then export the instance." If you try to programmatically build
   a design from scratch, you are using the API wrong.

2. **Brand Kit is the source of truth for visual identity.** Colors, fonts,
   logos, voice, image styles all live in Brand Hub (Enterprise) or the
   Brand Kit (Pro). Both the GUI and the API draw from it. Magic Studio
   reads it for AI generations. Brand Templates inherit it. Change Brand Kit,
   templates re-render with the new values. This is why EOS keeps brand
   identity in Canva and not duplicated in code.

3. **Magic Studio is GUI-only and is the moat.** Magic Design, Magic Media
   (Dream Lab on Leonardo Phoenix), Magic Switch, Magic Morph, Magic Edit,
   Magic Write, Background Remover, Brand Voice — none have Connect API
   equivalents. The API is for *scale*; the GUI is for *creative leverage*.
   The EOS pattern is: Antony uses Magic Studio in the GUI to design one
   beautiful Brand Template, then agents call autofill to make ten thousand
   variants of it. Never expect agents to be creative in Canva — that is
   not what the API surface is for.

The async job pattern (`POST` returns `{job: {id, status: in_progress}}`,
`GET /<resource>/<job_id>` returns the final state) is universal across
autofill, export, asset upload, design import, resize. Treat it as a single
skill: submit, exponential-backoff poll, handle terminal `success`/`failed`.

## Gotchas

- **Brand Template + Autofill require Canva Enterprise membership.** Pro
  and Free accounts get a 403 on `/v1/brand-templates` and `/v1/autofills`.
  For solo founder pre-revenue this means: either upgrade to Enterprise
  (~$30/user/mo, contact sales), use a manual GUI workflow until revenue
  justifies it, or co-design templates inside an existing Enterprise tenant.
  This is the single most common Connect API blocker.
- **September 2025 Brand Template ID migration.** Old IDs still work for
  6 months from migration; after that they 404. If you stored template
  IDs before Sept 2025, refetch via `GET /v1/brand-templates` and rewrite
  your config.
- **Scopes do not imply each other.** `asset:write` without `asset:read`
  means you can upload but cannot list. `design:content:write` does not
  grant `design:content:read`. Request both. Re-consent is required when
  scopes change — old refresh tokens don't auto-upgrade.
- **Autofill field types must match exactly.** A `text` field cannot accept
  an `image` payload, and vice versa. Read the dataset first, never guess
  field names. Capitalization matters. Missing required fields return 400
  with no auto-defaulting.
- **Image autofill takes `asset_id`, not URL.** Upload to
  `/v1/asset-uploads` first, wait for that job, then pass the resulting
  `asset.id` into the autofill payload's `image` field.
- **Export jobs are async, download URLs expire in 24 hours.** Persist the
  binary into your own storage (S3, R2, Neon BYTEA) immediately. Do not
  store the Canva URL in a database expecting to use it tomorrow.
- **Rate limits are per-endpoint, per-user, per-minute.** Notable:
  `POST /v1/exports` is 20/min/user, `GET /v1/exports/{id}` is 120/min/user.
  Autofill polling at 1s with exponential backoff stays well under the cap.
  A naive `while True: get; sleep(0.1)` loop will trip 429 in seconds.
- **PDF Print vs PDF Standard.** `format.type=pdf` defaults to standard;
  for print-shop output specify `pdf_export.export_quality=pro` and CMYK
  flags. Lyfe Spectrum print orders need this — don't send Standard PDFs
  to a printer.
- **Magic Studio is not in the API.** Do not promise an agent that it can
  call Magic Design or Dream Lab. Plan workflows around: Antony designs
  hero in GUI → save as Brand Template → agent autofills.
- **OAuth callback host must match exactly.** Tailscale IPs and `localhost`
  are accepted in dev, but production redirects must be HTTPS public
  domains. EOS uses a single canonical callback at the gateway, not
  per-service callbacks.
- **No webhooks for design lifecycle events.** As of 2026-04, Connect API
  has no design.created / design.updated webhooks. If you need to know
  when Antony manually edits a design, you must poll `GET /v1/designs`
  and diff `updated_at`.
- **Comments API is one-way useful.** You can create and read comments,
  but Canva surfaces them in the GUI without push notifications back to
  your integration — they are best used as audit trail, not realtime sync.
- **Free plan cannot use the Connect API at all** for production work
  beyond the most basic profile read. Treat Pro as the practical floor
  for non-autofill use, Enterprise as the floor for autofill.
- **`design.id` format is `DAF...` and stable**, but the `urls.edit_url`
  signed URL rotates. Store the design id, regenerate edit URLs on demand.
