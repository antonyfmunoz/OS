# composition/

System composition from registries, templates, capabilities, and mastery.
The layer that assembles executable plans from reusable components.

## Subdirectories

| Path | Purpose |
|------|---------|
| `mastery/` | Tool mastery (research, authoring, management) |
| `registries/` | Capability and component registries |

## §24 Reference

Canonical module tree §24: `composition/` — registries, templates,
capabilities, mastery, libraries.

## Boundary

Composition assembles but does NOT execute. It produces executable plans
that the execution layer carries out. The 15-step composition standard
(§11.6) is the target contract.
