## Shim Architecture

### Pattern: `sys.modules` (455 leaf modules)

```python
# Generated by r8d_generate_shims.py — do not edit
import runtime.{target} as _mod
import sys as _sys
_sys.modules[__name__] = _mod
```

This pattern makes the shim module literally *become* the canonical module
in Python's import system. Guarantees:
- `bridge_fn is canon_fn` (identity)
- Private names accessible (`_measure_pressure`, `_TERMINAL_STATES`)
- `mock.patch` on shim path affects canonical module
- Zero maintenance when canonical module changes

### Pattern: `import *` (2 package `__init__.py` files)

```python
# Generated by r8d_generate_shims.py — do not edit
from runtime.{package} import *  # noqa: F401,F403
```

Used for `eos_ai/transport/__init__.py` and `eos_ai/substrate/__init__.py`.
Package init files cannot use `sys.modules` replacement because Python's
import machinery interferes with child module resolution — replacing the
parent module in `sys.modules` causes child imports to resolve against the
canonical parent, bypassing the shim layer's `sys.modules` entries.

### Pattern: `empty_marker` (1 file)

`eos_ai/interfaces/__init__.py` — runtime has no `interfaces/__init__.py`,
so this is a placeholder allowing `from eos_ai.interfaces.X import Y`.

### Pattern: `package_marker` (1 file)

`eos_ai/runtime/__init__.py` — enables `from eos_ai.runtime.work_state`
and `from eos_ai.runtime.provider_state` depth-flattened imports.
