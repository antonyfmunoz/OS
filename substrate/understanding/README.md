# understanding/

Perception, interpretation, knowledge, and world modeling. Everything
that converts raw signals into structured understanding.

## Subdirectories

| Path | Purpose |
|------|---------|
| `domains/` | Domain-specific understanding |
| `embedding/` | Embedding generation and similarity |
| `intelligence/` | Intelligence gathering and analysis |
| `interpretation/` | Signal interpretation |
| `knowledge/` | Knowledge graph and domains |
| `ontology/` | Ontological primitives (PrimitiveObservation) |
| `patterns/` | Pattern recognition |
| `perception/` | Perception pipeline (ingestion orchestrator) |
| `reality/` | Reality modeling |
| `research/` | Research engine |
| `signals/` | Signal processing |
| `world_model/` | World model maintenance |
| `world_pulse/` | Real-time world state |

## §24 Reference

Canonical module tree §24: `understanding/` — perception, interpretation,
ontology, world_model, knowledge, patterns.

## Boundary

Understanding perceives and interprets. It does NOT execute actions or
make governance decisions. It feeds structured observations into state
and the control plane.
