# learning/

Feedback loops, skill evolution, self-modeling, and improvement over time.

## Subdirectories

| Path | Purpose |
|------|---------|
| `evolution/` | System evolution tracking |
| `feedback/` | Feedback loop processing |
| `self_model/` | Self-model maintenance |
| `skills/` | Skill learning and improvement |

## §24 Reference

Canonical module tree §24: `learning/` — feedback, evolution,
self_model, skills.

## Boundary

Learning observes outcomes and proposes improvements. It does NOT
directly modify system behavior — improvements flow through the
governance layer for approval before taking effect.
